# Sprint Contract 模板 — 车界管理后台

## 使用说明

在开始实现任何功能之前，填写本模板与 agent 协商范围。编码开始前确认验收标准。

---

## Sprint 概览

**功能名称**: [填写]
**预估会话数**: [1 / 2-3 / 3+]
**日期**: [YYYY-MM-DD]

## 角色

| 角色 | 职责 |
|------|------|
| **规划者** | 定义验收标准、范围边界、评分标准（在实现前） |
| **实现者** | 基于规划者的规格实现功能 |
| **评估者** | 对照验收标准和评分表审查实现（独立 agent 或人工审查） |

## 范围

### 范围内

- [ ] [具体可验证的行为描述]
- [ ] ...

### 范围外（明确排除）

- [ ] [不应在此 sprint 中实现的内容]
- [ ] ...

## 验收标准

使用 Checklist 格式，每项对应一个可执行验证命令或可观察行为：

1. [ ] [验收条件 1]
2. [ ] [验收条件 2]
3. [ ] ...
4. [ ] `npm run build` 构建通过
5. [ ] `bash scripts/check-architecture.sh` 通过
6. [ ] `npm run lint` 通过

## 示例：添加"导出 CSV"功能

### Sprint 概览

**功能名称**: 数据导出 CSV
**预估会话数**: 1
**日期**: 2026-06-11

### 范围

**范围内**:
- [ ] 每个 CRUD 页面的表格上方添加"导出 CSV"按钮
- [ ] 导出当前搜索结果的全部数据（不分页）
- [ ] CSV 文件命名格式：`{实体名}_{日期}.csv`

**范围外**:
- [ ] Excel (xlsx) 格式导出
- [ ] 自定义选择导出列
- [ ] 服务端导出

### 验收标准

1. [ ] Brand/Car/CarConfig/CarMedia/CarColor/CarRival/Persona/PersonaCar/Banner 九个页面均有导出按钮
2. [ ] 点击导出后，浏览器下载 CSV 文件
3. [ ] CSV 内容与表格列对应，中文列名
4. [ ] 导出所有匹配搜索条件的数据，不受分页限制
5. [ ] 访客模式下导出按钮可用（读操作）
6. [ ] `npm run build` 构建通过

### 验证计划

| 步骤 | 验证方法 | 预期结果 |
|------|---------|---------|
| 1 | 打开 Brand 页面，点击导出 | 下载 `品牌_20260611.csv` |
| 2 | 搜索品牌名后导出 | CSV 仅含匹配结果 |
| 3 | 访客登录后点击导出 | 正常下载 |
| 4 | `npm run build` | 构建成功 |
