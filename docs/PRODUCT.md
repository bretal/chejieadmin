# 产品描述 — 车界管理后台

## 这是什么？

车界管理后台是一个面向汽车行业的数据管理 SPA。管理员可以对品牌、车型、配置、媒体、颜色、竞品关系、用户画像和 Banner 进行完整的 CRUD 操作。系统内置 RAG 问答助手，支持上传文档后基于知识库进行 AI 对话。提供访客模式让未注册用户以只读方式浏览数据。

## 核心功能

### 作品展示页（首页）
- 个人技术能力展示：Hero 区、技术栈轮播、项目案例（Web/小程序/后端）
- 代码片段展示
- 滚动视差和视口交叉动画

### 认证与权限
- 用户名密码登录
- 访客免密登录（只读模式）
- Token 基于 localStorage 持久化，自动附加到 API 请求
- 401 自动清除凭据并重定向登录页

### 访客模式
- 访客登录后可查看所有管理页面
- 所有写操作按钮（新增/编辑/删除）disabled 并显示"访客模式下无操作权限"
- Header 显示橙色"访客模式"标签

### 管理后台布局
- Sider（200px 可折叠）+ Header（64px）+ Content
- 玻璃拟态风格：半透明背景 + 毛玻璃模糊 + 柔和阴影
- 菜单项带图标，根据当前路径自动高亮

### 品牌管理
- 品牌列表分页展示（ID、名称、Logo、国家、首字母、排序、状态）
- 名称模糊搜索
- 新增/编辑表单（名称、状态、Logo URL、国家、首字母、简介、排序）
- 单个/批量删除

### 车型管理
- 车型列表，关联品牌，含车身类型、能源类型、尺寸参数
- 标准 CRUD 操作

### 配置管理
- 车型下的配置（车款）管理：价格、续航、马力、配置项、芯片型号、音响品牌等
- 标准 CRUD 操作

### 媒体管理
- 车型关联的图片/视频管理：媒体类型、URL、排序
- 标准 CRUD 操作

### 颜色管理
- 车型可选颜色：颜色名称、色值、图片 URL
- 标准 CRUD 操作

### 竞品关系管理
- 车型间竞争关系：选择竞品车型和竞争类型
- 标准 CRUD 操作

### 用户画像管理
- 画像创建和管理：名称、描述、标签、特征参数
- 标准 CRUD 操作

### 画像推荐管理
- 用户画像到车型的推荐关系
- 标准 CRUD 操作

### Banner 管理
- 推广 Banner：标题、图片 URL、链接、排序、启用/停用
- 标准 CRUD 操作

### RAG 问答助手（需认证）
- 左侧面板：文档上传（PDF/DOCX/MD/CSV/TXT）、文档列表、删除
- 右侧面板：对话界面，SSE 流式响应，打字机效果
- 访客限制上传一个文档
- 对话历史持久化

### RAG 问答助手（公开）
- 无需登录即可使用
- 浮动粒子动效背景
- 功能同认证版 RAG 助手

### 前端监控
- Web Vitals 性能指标采集（FCP/LCP/CLS/INP）
- 导航时序（DNS/TCP/TLS/TTFB/白屏时间）
- 全局 JS 错误和 Promise 拒绝捕获
- React ErrorBoundary 组件级错误捕获
- 自动点击埋点 + 路由变更追踪
- 批量上报（指数退避重试）+ 离线队列 + sendBeacon
- Sourcemap 上传（生产构建自动上传后删除）

### Docker 部署
- 多阶段构建（Node 构建 + Nginx 运行）
- SSL 支持（Let's Encrypt）
- API 反向代理
- SPA 路由回退

## 用户角色

| 角色 | 权限 |
|------|------|
| 管理员 | 完整 CRUD 权限 |
| 访客 | 只读浏览，RAG 限上传 1 个文档 |

## 约束

- 前端纯静态 SPA，无 SSR
- 后端 API 由独立的 Java + Python 服务提供
- 不需要离线支持（无 Service Worker）
- 玻璃拟态主题仅在支持 `backdrop-filter` 的浏览器中完整显示
